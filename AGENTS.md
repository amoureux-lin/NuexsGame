# AGENTS.md

## Cursor Cloud specific instructions

### Project Overview

NexusGame is a multi-game casino/card game client built with **Cocos Creator 3.8.8** and TypeScript. It uses a custom **Nexus Framework** (delivered as a Cocos Creator Extension at `extensions/nexus-framework/`). The architecture is lobby + sub-games (Tongits, SlotGame), loaded as separate Cocos Bundles.

### Dependencies

Two `npm install` locations are required:
- `/workspace` — installs `protobufjs`
- `/workspace/extensions/nexus-framework/` — installs `@cocos/creator-types`, `@types/node`, `typescript`

After installing, build the extension: `npm run build` in `extensions/nexus-framework/`.

### TypeScript Compilation

- **Extension (`extensions/nexus-framework/source/`)**: Can be type-checked independently via `npx tsc --noEmit` in that directory.
- **Game scripts (`assets/script/`)**: Use Cocos Creator's `db://nexus-framework/` module resolution and `cc` engine imports. They **cannot** be type-checked outside the Cocos Creator editor because `temp/tsconfig.cocos.json` (generated by the editor) is not committed.

### Lint / Test

- No ESLint, Prettier, or other linter is configured in this project.
- No automated test framework (jest, vitest, etc.) is configured.
- The extension build (`npm run build` in `extensions/nexus-framework/`) serves as the primary static verification.

### Running the Application

This project requires **Cocos Creator 3.8.8** (a GUI IDE) to preview/build. The editor is not available in this cloud environment. Key points:

- The game supports `?mock=true` URL parameter to skip all network connections and use local mock data.
- Without the Cocos Creator editor, the game cannot be previewed or built to HTML5.
- See `assets/README-BUNDLE.md` for bundle configuration and `assets/README-FLOW.md` for game flow documentation.
- The Nexus Framework documentation is at `extensions/nexus-framework/nexus-framework.md`.

### Networking

- Backend HTTP API: `https://gwm.herondev.xin` (debug mode)
- WebSocket gateway address is obtained from `/api/v1/get_game_config`
- In debug/localhost mode, a token is auto-generated via `/debug/enter_game`
